SOAL 2: client → urls.py → views.py → models.py / template → response → client, kaitannya: urls.py → berfungsi sebagai “peta jalan” yang menentukan ke fungsi mana request dari client akan diarahkan.
views.py → berperan sebagai “otak” yang menjalankan logika aplikasi, memproses data, dan menyiapkan informasi untuk ditampilkan.
models.py → menyimpan struktur data dan berinteraksi dengan database; views mengambil atau menyimpan data di sini jika diperlukan.
Template HTML → bertugas menampilkan hasil olahan views dalam bentuk halaman web yang bisa dilihat user.
Client / Browser → sebagai pengirim request dan penerima response HTML dari server.

SOAL 3: File settings.py berfungsi sebagai pusat pengaturan proyek Django. Di dalamnya terdapat konfigurasi seperti daftar aplikasi yang aktif, pengaturan database, lokasi file statis, path template, serta aspek keamanan seperti secret key, mode debug, dan allowed hosts. Selain itu, file ini juga mengatur berbagai konfigurasi yang dibutuhkan saat deployment. Singkatnya, hampir semua pengaturan penting proyek Django dikendalikan melalui settings.py
SOAL 4: Di Django, migrasi database dilakukan dengan cara menyesuaikan struktur database dengan model yang ada di models.py. Jika ada perubahan pada model—misalnya menambahkan kolom baru, membuat tabel baru, atau menghapus kolom—Django akan membuat file migrasi yang berisi langkah-langkah untuk memperbarui database. File migrasi ini kemudian dijalankan agar database sesuai dengan model yang terbaru, sehingga kode dan database tetap sinkron.
SOAL 5: Django dipilih karena menyediakan banyak fitur bawaan, seperti autentikasi, ORM, dan template engine, sehingga mempermudah pemula untuk belajar. Framework ini juga menerapkan pola desain MTV (Model Template View) yang terstruktur, sehingga mahasiswa lebih mudah memahami alur request response dalam pengembangan web. Selain itu, dokumentasinya lengkap dan didukung komunitas yang besar, membuat Django ideal sebagai langkah awal untuk belajar pengembangan perangkat lunak berbasis framework
